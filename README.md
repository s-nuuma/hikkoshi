# 同棲準備・引越し 統合ダッシュボード (Hikkoshi Dashboard)

同居を始める二人が、スマートフォンのブラウザで快適にタスクを管理し、初期費用を調整・分担できる1ファイルのレスポンシブWebアプリケーション（SPA）です。

**本番公開URL: [https://s-nuuma.github.io/hikkoshi/](https://s-nuuma.github.io/hikkoshi/)**

## 主な機能

1. **📊 ダッシュボード (進捗状況)**
   - 全体のタスク完了率をアニメーション付きのSVG円グラフで表示。
   - 愛翔、竣介それぞれの進捗状況をプログレスバーで視覚化。
   - 直近で対応すべき優先未完了タスク（最大5件）をリスト表示し、ダッシュボードから直接ステータス変更が可能。

2. **📋 タスク管理**
   - 提供されたCSVタスク一覧（全51件）を忠実に再現。
   - カテゴリ別のアコーディオン表示、および検索クエリ・ステータス・担当者による柔軟なフィルタリング。
   - 愛翔、竣介それぞれのステータス（未対応 / 進行中 / 済 / やらない）をインタラクティブに変更可能。

3. **💰 お金シミュレーター**
   - 初期費用（家賃、敷金、礼金、仲手、鍵交換など）の金額を自由に変更可能。
   - 一括負担比率調整スライダー（愛翔: 0%〜100% / 竣介: 100%〜0%）で各自の負担金額を直感的に調整。
   - 項目ごとの個別調整スライダーにも対応。
   - CSS/SVGアニメーションによる負担比率スタックバーで、各自の負担金額を分かりやすくビジュアル化。

4. **💡 お役立ちガイド**
   - 転出届の提出手順、ライフライン（電気・ガス・水道・ネット）の停止/開始手続き、当日の傷チェック、新居の住所登録（世帯主の扱い）などの具体的な手順・持ち物・注意点を整理。

5. **🔄 LINE簡単同期 (データ同期)**
   - データベース（Firestore等）を必要としないスタンドアロン構成。
   - 現在のタスクチェック状況、金額、負担比率の全設定値を1クリックで暗号化（JSON/Base64）し、クリップボードにコピー可能。
   - 相手から受け取った文字列を貼り付けるだけで、即座に同じ状態を復元・同期できるインポートフォームを搭載。
   - `localStorage` への自動保存機能を備え、ページがリロードされても状態は消えずに永続化。

## 技術スタック

- **フロントエンド**: HTML5, TypeScript, Tailwind CSS (Tailwind CLI / CDN), Vue.js 3 (Global Build CDN)
- **ビルド環境**: Vite, Node.js
- **テストツール**: Vitest (単体テスト), Playwright (E2Eテスト)

## ディレクトリ構造

```text
├── index.html          # ビルド成果物（完全にスタンドアロンで動作する単一HTMLファイル）
├── src/
│   ├── index.html      # 開発用のHTMLテンプレート
│   ├── main.ts         # メインのアプリケーションロジック（Vue 3 アプリ）
│   ├── utils.ts        # 初期データ定義、費用計算、同期用のエンコード/デコード処理
│   ├── types.ts        # 型定義ファイル
│   └── style.css       # Tailwind CSS ディレクティブとカスタムスタイル
├── tests/
│   ├── app.test.ts     # 単体テスト (Vitest)
│   └── e2e.spec.ts     # E2Eテスト (Playwright)
├── build.js            # HTML、CSS、JSをマクロ置換で1ファイル化するカスタムビルドスクリプト
├── vite.config.ts      # Vite設定ファイル
├── tailwind.config.js  # Tailwind CSS設定ファイル
├── postcss.config.js   # PostCSS設定ファイル
├── tsconfig.json       # TypeScriptコンパイル設定ファイル
└── playwright.config.ts# Playwright設定ファイル
```

## 開発と実行方法

### 依存関係のインストール

WSLまたはローカル環境で実行します：

```bash
npm install
```

### 開発用サーバーの起動

```bash
npm run dev
```

### プロダクションビルドの生成

ビルドを実行すると、プロジェクトルートに完全に1つのファイルに統合された `index.html` が出力されます。

```bash
npm run build
```

### テストの実行

#### 単体テスト (Vitest)

```bash
npm run test
```

#### E2Eテスト (Playwright)

```bash
npm run test:e2e
```

## テストステータス

- **Vitest (単体テスト)**: 計算ロジックや同期Base64処理など 6件すべてパス完了。
- **Playwright (E2Eテスト)**: モバイル画面エミュレーションでのタブ切り替え・データ更新・シミュレーター連動など 4件すべてパス完了。
